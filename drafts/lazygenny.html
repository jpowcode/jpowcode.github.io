LazyGenny

Mon 27 November 2017
By jpowcode

I'm a mathematician originaly and sequences of numbers have always fasinated me. For example there are the happy numbers You can find if a number is happy by taking its digits, squaring them and adding them to get a new number. Take the sum suare of the digits of the new number and repeat. If this ends in a 1, the number is said to be happy. For example, take 19

$$ 1^2 + 9^2 = 82 $$ $$ 8^2 + 2^2 = 68 $$ $$ 6^2 + 8^2 = 100 $$ $$ 1^2 + 0^2 + 0^2 = 1 $$

So 19 is a happy number.

I wanted a library that could answer questions such as "which of the happy numbers are also prime, or perfect numbers" I also wanted the sequences to be laziliy generated as an when I wanted them. So, for example, I might ask for the sequence of prime numbers but they will only be calculated if and when I ask for a particular prime number, say the 100'th. Then if I ask for the 99'th I should not have to wait for the sequence to be regenerated. I also wanted a nice syntax for my library to allow me to chain multiple sequeces together. I couldn't find anything that did what I wanted so I decided to write my own. The O'Reilly book Functional Programming in Python gave me a good head start to get going with lazy evaluation. LazyGenny just extends that approach a bit.

So to start with I define a new data structure called Seq. This basically holds a list as its data, but the difference being that all it's methods return a new list rather than modifying the existing list. So calling the append method will return a new list with the item added on the end.
